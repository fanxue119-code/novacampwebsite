# Nova 官网项目推进手册

> 本文档为 Nova 官网项目三方（大雪 / 饭团 / 技术 / 海报设计）共同对齐的推进说明。
> 上线目标：2026 年 5 月中下旬（W7 末）

---

## 一、四方分工

**大雪**
项目经理 + 原型 + UI。负责需求管理、Figma 原型、高保真 UI 设计稿、阶段验收、整体进度协调。

**饭团**
业务方。负责业务规则（SKU 价格、拼团、分销、退款）、内容素材（老师资料、课表、文案）、上线决策。

**技术**
开发方。负责官网 H5、后台管理、核销端的开发与部署。前后端内部分配，对接入口为技术负责人。

**海报设计**
视觉物料。全程参与，负责官网内视觉素材（KV、Banner、老师头图等）与上线推广物料。视觉风格与 UI 设计稿保持一致。

---

## 二、阶段总览

| 阶段 | 时间 | 目标 |
|---|---|---|
| Phase 0 · 准备 | W0（本周） | 环境就绪 + 需求冻结 + 前置审批启动 |
| Phase 1 · 静态页+表单 | W1–W2 | 官网能打开、能填报名表 |
| Phase 2 · 订单+支付+核销 | W3–W4 | 能下单、付钱、扫码核销 |
| Phase 3 · 次卡 | W5 上半 | 次卡能卖，按老师核销 |
| Phase 4 · 拼团+分销 | W5 下半–W6 | 团池、分销链接、自动归团 |
| Phase 5 · 上线 | W7 | 联调、压测、培训、全量发布 |

## 三、三条关键死线

- **W0 结束前**：启动支付宝商户号申请（审批约 2 周，不启动将阻塞 Phase 2）
- **W2 结束前**：饭团交付次卡完整价格表（不交付将阻塞 Phase 3）
- **W4 结束前**：饭团交付拼团完整规则（不交付将阻塞 Phase 4 和上线）

---

## 四、各阶段分工

### Phase 0 · 准备（W0）

**目标**：环境就绪，需求冻结，前置审批启动。

**大雪**
- 发出本手册 + 技术附录，组织 PRD 核对会
- Figma 原型定稿
- 与海报设计对齐视觉基调
- 整理外网参考链接发技术

**饭团**
- 确认业务规则清单 + 各项交付时间
- 启动阿里云服务器 / OSS / CDN 采购
- 准备老师资料（下阶段交付）
- **启动支付宝商户号申请**

**技术**
- 评估服务器方案，输出配置建议
- 输出接口清单初稿 + 数据库 ER 图 + 开发排期

- 确认域名 + ICP 备案

**海报设计**
- 对齐视觉基调（色系、字体、Logo 规范）
- 启动首页 KV 提案

**阶段交付**
- Figma 原型定稿
- 技术接口清单 + ER 图 + 排期表
- 饭团业务规则交付时间承诺
- 支付宝商户号申请回执
- 海报设计视觉基调稿

---

### Phase 1 · 静态页+表单（W1–W2）

**目标**：官网能打开、能看老师和课程、能填报名表。

**大雪**
- 完成所有页面高保真 UI 设计稿
- 交付切图 + 设计规范给技术
- W1 末中期走查，W2 末阶段验收

**饭团**
- W1 交付：老师资料（照片/简介/视频）、完整课表、FAQ、报名须知
- W2 交付：退款政策文案、英文翻译文案
- **W2 结束前**：次卡完整价格表 + 组合方式 + 全科全套促销规则

**技术**
- 项目框架搭建（中英双语 i18n）
- 首页 Landing（含视频/动效占位）
- 老师详情 / 课表 / FAQ / 报名须知页面
- 报名表单页 + 字段校验 + 协议
- 后台内容配置模块

**海报设计**
- 首页 KV 主视觉
- 老师头图组（含 hover 效果）
- 课程卡片视觉素材

**阶段交付**
- 官网前台（静态展示 + 报名表单）可用
- 后台内容配置可用
- 视觉素材入库

---

### Phase 2 · 订单+支付+核销（W3–W4）

**目标**：跑通"选课 → 下单 → 付钱 → 拿二维码 → 现场扫码入场"闭环。

**大雪**
- 设计支付相关页面 UI（支付选择、支付中、支付成功、我的订单）
- 设计核销端 UI
- 与技术走查支付异常场景（超时、取消、重复）
- W4 末阶段验收（真实走一单支付宝 + Wise）

**饭团**
- 提供 Wise 收款账户信息
- 指定 Wise 人工确认员 + 约定确认频率
- **W4 结束前**：拼团完整规则 + 分销商名单 + 返现规则

**技术**
- 订单系统（完整状态机）
- 支付宝 H5 支付 + 异步通知
- Wise 海外支付流程（用户上传凭证 + 后台人工确认）
- 报名成功页 + 核销二维码
- 核销端（登录、扫码、确认、防重复）
- 后台订单 / 支付 / 核销管理 + 数据导出

**海报设计**
- 报名成功页视觉
- 分享卡样式

**阶段交付**
- 下单 → 支付 → 核销闭环跑通
- 国内（支付宝）+ 海外（Wise）两套支付可用
- 核销端可用

---

### Phase 3 · 次卡（W5 上半）

**目标**：次卡能卖，按老师核销。

> 依赖：饭团 W2 已交付次卡规则。

**大雪**
- 设计次卡 SKU 展示页 UI
- 设计"选老师 + 选课程"交互
- 设计"我的次卡"页面
- W5 中验收

**饭团**
- 提供 700 元档老师名单
- 确认次卡适用老师范围
- 次卡运营策略（是否允许退、有效期）

**技术**
- 次卡 SKU 配置（1/4/5/10 次 + 组合包）
- 购买时绑定老师 + 课程
- 核销按老师 × 课次扣减
- 后台次卡余量 + 按老师统计

**海报设计**
- 次卡产品卡视觉
- 组合包促销素材

**阶段交付**
- 次卡可购、可用、可核销
- 后台次卡数据可查

---

### Phase 4 · 拼团+分销（W5 下半–W6）

**目标**：拼团名额自动分配，分销链接可追踪。

> 依赖：饭团 W4 已交付拼团规则。

**大雪**
- 设计拼团页 UI（名额、成员、分享）
- 设计分销落地页 UI
- 与饭团确认分销返现流程（系统自动 / 线下人工）
- W6 末阶段验收（跑真实拼团，验自动归团）

**饭团**
- 分销商最终名单 + 专属链接分配
- 返现规则细则（金额 / 比例 / 结算时间）
- 团满后运营话术

**技术**
- 团池模型（200 × 20 人团 + 200 × 10 人团）
- 分销链接系统 + 购买归属记录
- 自动填团（先填 1 号，满了填 2 号）
- 团满自动关档
- 后台拼团 / 分销数据报表

**海报设计**
- 拼团活动主视觉
- 分销物料（朋友圈图、海报、分享卡）

**阶段交付**
- 拼团功能完整可用
- 分销链接可追踪，数据可查

---

### Phase 5 · 上线（W7）

**目标**：联调、压测、培训、发布。

**大雪**
- 组织全链路走查
- 组织压测观摩
- 准备上线公告文案
- 灰度 → 全量发布节奏把控

**饭团**
- 确认上线时间窗口
- Wise 人工确认员就位
- 现场核销员培训排期

**技术**
- 全链路联调测试
- 压测（单场 800 人并发核销）
- 后台使用培训（运营 + 财务）
- 核销端使用培训（安保）
- 灰度 → 全量发布
- 上线后值守 48 小时

**海报设计**
- 上线推广物料（朋友圈图、公众号头图、社交海报）
- Plan B 物料备用

**阶段交付**
- 官网正式上线
- 运营团队培训完毕
- 推广物料到位

## 六、协作节奏

| 频次 | 动作 |
|---|---|
| 每周一 | 大雪 × 技术：上周完成 + 本周目标 + 卡点（30min）|
| 每周三 | 大雪 × 海报设计：本周素材需求对齐 |
| 每周三 | 大雪 × 前端：UI 走查 |
| 每周五 | 大雪 × 饭团：饭团本周交付 + 下周需求（30min）|
| 每周五 | 大雪发周报（完成 / 进行中 / 风险）|

---

## 七、风险与应对

| 风险 | 应对 |
|---|---|
| 饭团规则延期 | 先推进不依赖规则的阶段，规则到位再启动对应 Phase |
| 支付宝商户号审批慢 | W0 启动，每周追进度 |
| Wise 人工确认流程断链 | W3 前定 SOP，指定专人 |
| 首页视频/动效影响加载 | 压缩 + CDN + 首屏占位图 |
| 现场 800 人并发核销 | 压测 + 核销端离线队列 |
| 海报设计与 UI 不一致 | W0 对齐视觉基调，以 UI 稿为准 |
